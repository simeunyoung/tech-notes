## 전체 흐름

```
1. Vercel에 도메인 추가
2. www 도메인도 추가
3. 가비아 DNS 설정 (A 레코드 + CNAME)
4. Vercel에서 Valid Configuration 확인
```

---

## 1. Vercel에 도메인 추가

Vercel 프로젝트 → Settings → Domains → Add

도메인 종류에 따라 입력 방식이 다르다.

**일반 도메인**은 그대로 입력한다.

```
homepage.com
```

**한글 도메인**은 퓨니코드(Punycode)로 변환 후 입력해야 한다.
[punycoder.com](https://www.punycoder.com)에서 변환할 수 있다.

```
홈페이지.com → xn--hu5b25b77nvwc.com
```

브라우저 주소창에는 한글로 보이지만 실제 DNS에서 처리되는 주소는 퓨니코드 형식이다.

---

## 2. www 도메인도 추가

루트 도메인만 추가하면 `www.도메인`으로 접속했을 때 연결이 안 된다.
www 도메인도 함께 추가해야 한다.

```
www.homepage.com
www.homepage.kr
www.xn--939au0g56v.com
```

www 도메인을 추가하면 Vercel이 루트 도메인 → www 자동 리다이렉트를 함께 생성해준다.

---

## 3. 가비아 DNS 설정

My가비아 → DNS 관리 → 도메인 선택 → DNS 설정

아래 두 가지 레코드를 추가한다.

| 타입  | 호스트명 | 값                   |
| ----- | -------- | -------------------- |
| A     | `@`      | Vercel에서 확인한 값 |
| CNAME | `www`    | Vercel에서 확인한 값 |

**A 레코드** — 루트 도메인(`@`)을 Vercel IP로 연결한다.

**CNAME 레코드** — www 서브도메인을 Vercel 도메인으로 연결한다.
값은 도메인마다 다르므로 Vercel에서 직접 확인해야 한다.

### CNAME 값 확인 방법

Vercel → 프로젝트 → Settings → Domains → 해당 도메인 클릭

DNS Records 섹션의 **ALIAS** 항목 값이 CNAME에 넣을 값이다.

```
fdbc422123123ddd.vercel-dns-014.com.
```

> 끝에 `.` 이 붙어 있다. 가비아 입력 시 그대로 붙여넣어야 한다. 없으면 동작하지 않는다.

---

## 4. 완료 확인

Vercel Domains 목록에서 모든 도메인이 ✅ **Valid Configuration** 상태가 되면 완료다.

| 상태                                   | 의미                  |
| -------------------------------------- | --------------------- |
| ✅ Valid Configuration + Production    | 메인 도메인           |
| ✅ Valid Configuration + 307/308 → www | 루트 → www 리다이렉트 |

---

## 주의사항

**CNAME 값 끝에 `.` 필수**
가비아 입력 시 `vercel-dns.com.` 처럼 마지막 점까지 포함해서 입력해야 한다.
루트 도메인의 끝을 의미한다.

**한글 도메인은 퓨니코드 변환 필수**
Vercel과 DNS 모두 한글 도메인을 직접 처리하지 않는다.
[punycoder.com](https://www.punycoder.com)에서 변환 후 입력한다.
퓨니코드란? DNS 시스템이 영어/숫자/하이픈만 이해할 수 있어서, 한글 도메인을 DNS가 읽을 수 있는 영문으로 변환한 것이다.

**DNS 전파 시간**
설정 후 바로 적용되지 않을 수 있다. 보통 수 분에서 수 시간, 최대 48시간 소요된다.

**이미 다른 프로젝트에 등록된 도메인**
Vercel은 도메인을 하나의 프로젝트에만 연결할 수 있다.
다른 프로젝트에 이미 등록된 도메인은 먼저 제거한 후 추가해야 한다.
